# Monstrinhomon — Pacote de Execução MVP 0.3

Status: pacote operacional para execução técnica  
Milestone: **MVP 0.3 — Wild Loop Estável**

## Finalidade

Este pacote transforma os blocos de documentação estratégica do Monstrinhomon em um plano de execução prático para o próximo avanço do projeto.

O objetivo deste ciclo **não é expandir o jogo**. O objetivo é estabilizar o menor fluxo jogável, salvável e repetível.

## Fluxo mínimo do MVP 0.3

```text
Criar jogador
→ escolher classe
→ receber starter correto
→ iniciar encontro wild
→ usar ataque/carta simples
→ capturar ou derrotar
→ ganhar recompensa
→ salvar
→ continuar depois
```

## Arquivos deste pacote

| Arquivo | Função |
|---|---|
| `MVP_0_3_WILD_LOOP_ESTAVEL.md` | Define escopo, fora de escopo, critérios de pronto e falha. |
| `BACKLOG_GITHUB_ISSUES_PRIORIZADO.md` | Lista issues práticas para execução no GitHub. |
| `ROTEIRO_AUDITORIA_PRATICA.md` | Guia para testar o jogo real antes/depois de alterações. |
| `PROMPT_MESTRE_CODEX_COPILOT.md` | Prompt seguro para Codex/Copilot implementar uma issue por vez. |
| `CHECKLIST_FINAL_EXECUCAO.md` | Checklist para declarar o MVP 0.3 concluído. |

## Regra de ouro

```text
Uma issue por vez. Um fluxo por vez. Um teste por vez.
```

Evite pedir para qualquer IA “arrumar o projeto todo”. Isso aumenta risco de regressão, duplicação de regra e quebra de save.

## O que está temporariamente fora de escopo

Até o Wild Loop estar estável, evitar:

- deckbuilding completo;
- batalha em grupo avançada;
- boss complexo;
- campanha longa;
- dashboard terapêutico robusto;
- economia avançada;
- novos mapas grandes;
- novas linhas evolutivas;
- refactor total do projeto.
